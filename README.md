# AudioInputIcon

A tiny macOS menu bar app that shows which audio input device is currently active. Switch your mic and the icon updates instantly.

## Why would you use this?

If you have multiple audio input devices (e.g. your MacBook's built-in mic, AirPods, USB mic, etc), it's easy to lose track of which one is currently active, especially when they get connected/disconnected frequently. This app gives you a quick visual indicator in the menu bar, so you can be sure you're using the right mic.

## Installation

Download the latest release from the [Releases](https://github.com/alcuadrado/AudioInputIcon/releases) page, decompress the `.tgz` file, and move `AudioInputIcon.app` to your `/Applications` folder.

## Icon mapping

| Device               | Match method                                                                   | Icon               |
| -------------------- | ------------------------------------------------------------------------------ | ------------------ |
| MacBook built-in mic | Name contains "macbook", "built-in", "internal", or transport type is built-in | `laptopcomputer`   |
| AirPods Max          | Name contains "airpods max"                                                    | `airpodsmax`       |
| AirPods Pro          | Name contains "airpods pro"                                                    | `airpodspro`       |
| AirPods              | Name contains "airpods"                                                        | `airpods`          |
| Beats                | Name contains "beats"                                                          | `beats.headphones` |
| Bluetooth headphones | Bluetooth transport type                                                       | `headphones`       |
| Webcam / FaceTime    | Name contains "webcam", "facetime", "camera"                                   | `web.camera`       |
| External display     | Name contains "display", "monitor", "hdmi", or DisplayPort/HDMI transport      | `display`          |
| USB / studio mic     | Name contains "yeti", "rode", "shure", etc., or USB transport                  | `mic.fill`         |
| EarPods              | Name contains "earpods", "earbud"                                              | `earpods`          |
| Virtual device       | Virtual transport type                                                         | `mic.badge.xmark`  |
| No device            | No input device available                                                      | `mic.slash`        |
| Unknown              | Fallback                                                                       | `mic`              |

## Features

- Zero runtime dependencies — only uses native macOS APIs
- Updates in real-time via CoreAudio property listener
- Optional "Start at Login" via SMAppService
- No microphone access required — only reads device metadata

## Building

AudioInputIcon is written in Zig and cross-compiled from Linux targeting `aarch64-macos`, so that you don't need to install any toolchain on your host, apart from the [Command Line Tools](https://developer.apple.com/documentation/xcode/command-line-tools).

> You can get away without installing the Command Line Tools if you copy a macOS SDK from another machine.

### Prerequisites

- **Linux**: [Zig 0.13.x](https://ziglang.org/download/)
- **macOS SDK**: Copy a macOS SDK (13.0+) from a Mac with Xcode or Command Line Tools installed. The SDK lives at:

  ```
  /Library/Developer/CommandLineTools/SDKs/MacOSX*.sdk
  ```

  Make sure not to copy a symlink!

### Build process

The build process consists of three steps:

#### Compile on Linux with the macOS SDK

```bash
export MACOSX_SDK_PATH=/path/to/MacOSX.sdk
zig build -Dsysroot=$MACOSX_SDK_PATH
```

#### Generate the app icon

The app icon is generated by a separate Zig tool (`IconGen`) because SF Symbols can only be rendered on macOS at runtime.

Copy `zig-out/bin/IconGen` to your Mac and run it. Then copy the generated `AppIcon.icns` back to the Linux machine:

```bash
./IconGen
# mv AppIcon.icns root-of-the-project-in-linux/bundle/
```

#### Bundle the app on Linux

Once the icon is generated, and available as `bundle/AppIcon.icns`, you can create the final `.app` bundle on Linux by running:

```bash
./scripts/bundle.sh
```

This will create `AudioInputIcon.app`, which you can copy to `/Applications` on your Mac.

#### Troubleshooting

**Note on overwriting binaries**: If you overwrite a Mach-O binary in-place on macOS (e.g. via a shared mount), the OS may kill it on next launch. Always `rm` the old binary first, then copy the new one.
